# Code Template

A lightweight webpack starter for The Odin Project JavaScript projects.

## What this template includes

- webpack split into common, development, and production configs
- webpack-dev-server for local development
- HTML generation with `html-webpack-plugin`
- CSS imports through `style-loader` and `css-loader`
- image/font asset handling through webpack asset modules
- Prettier formatting scripts
- basic ESLint flat config for browser-side JavaScript
- GitHub Pages deployment through the `dist` subtree

## Install

```bash
npm install
```

## Development

```bash
npm run dev
```

or:

```bash
npm start
```

Both run webpack-dev-server using `webpack.dev.js`.

## Production build

```bash
npm run build
```

The production build is generated into `dist/`. Do not edit `dist` by hand. It is generated output, not source code. Yes, apparently we need to keep telling humans not to edit the printer output.

## Formatting and linting

```bash
npm run format
npm run format:check
npm run lint
```

Use `format` when you want Prettier to rewrite files. Use `format:check` before committing if you only want to check formatting.

## Deploy to GitHub Pages

```bash
npm run deploy
```

The `predeploy` script runs `npm run build` first, then the `deploy` script pushes the `dist` folder to the `gh-pages` branch.

In GitHub Pages settings, use:

- Source: Deploy from a branch
- Branch: `gh-pages`
- Folder: `/root`

## Webpack reminder

- `src/` is where source code lives.
- `dist/` is generated by webpack.
- `webpack.common.js` contains shared config.
- `webpack.dev.js` is for local development.
- `webpack.prod.js` is for production builds.
- loaders transform imported files like CSS.
- plugins add extra build behavior like generating HTML.
